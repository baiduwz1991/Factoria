Overview

Factoria mod API 0.1 supports local loose-folder, data-and-resource mods. Mods can add or replace terrain definitions, terrain visuals, terrain autoplace rules, and planet presets. Script mods are not supported in API 0.1.

The game executable is the runtime. The built-in content is the always-enabled core mod. Player mods load after core.

概览

Factoria Mod API 0.1 支持本地松散文件夹形式的数据/资源型 mod。mod 可以新增或整条替换地形定义、地形视觉、地形自动生成规则和星球预设。API 0.1 暂不支持脚本 mod。

游戏本体负责运行时逻辑。内置内容会作为永远启用的 core mod 加载，玩家 mod 总是在 core 之后加载。

Mod-first Workflow

New content ideas should usually start as an experiment mod. This keeps data iteration fast and forces the engine API to stay honest about what it can support.

StepDecision
ExperimentCreate the idea under user://mods/my_experiment_mod.
IterateUse the manifest, JSON data files, and resource images/audio/icons.
ShipIf it works as optional content, release it as an independent official mod.
PromoteIf it should become default content, merge it into assets/data/core.
ExtendIf the mod exposes a missing system capability, extend the base game API instead of hard-coding the content.

Good mod-first domains

  • Terrain and planet presets
  • Resource distribution
  • Items, recipes, building parameters, and tech tree data
  • Decorations
  • Audio, textures, and icons
  • Future quest and story data

Do not force mod-first

  • Chunk streaming algorithms
  • Core save format logic
  • The C# terrain rendering algorithm itself
  • Controller lifecycle
  • UIManager and SceneManager
  • Performance-critical simulation kernels
  • Network synchronization rules, if multiplayer is added later

Mod-first 工作流

新的内容想法通常应先作为实验 mod 开始。这样内容迭代会更快，也能反过来检验本体 API 是否足够表达这些设计。

阶段决策
实验新想法先做成 user://mods/my_experiment_mod。
迭代使用 manifest、JSON 数据文件和资源图片/音频/图标快速调整。
发布满意后，如果它适合作为可选内容，就作为独立官方 mod 发给玩家。
转正如果它应该成为默认官方内容，就合并进 assets/data/core。
扩展如果某个 mod 暴露出当前系统能力不够，应扩展本体 API，而不是把内容硬塞进代码。

适合 mod-first 的内容

  • 地形和星球预设
  • 资源分布
  • 物品、配方、建筑参数和科技树
  • 装饰物
  • 音效、贴图和图标
  • 后续任务/剧情数据

不适合直接 mod-first 的内容

  • 区块流送算法
  • 存档格式核心逻辑
  • C# 地形渲染算法本身
  • Controller 生命周期
  • UIManager / SceneManager
  • 性能关键模拟内核
  • 网络同步规则，如果以后做多人

Folder Layout

Put local mods under the Godot user data mods directory. Each mod folder must contain factoria.mod.json.

目录结构

本地 mod 放在 Godot 用户数据目录的 mods 文件夹下。每个 mod 文件夹都必须包含 factoria.mod.json。

user://mods/
  active_mods.json
  author.my_mod/
    factoria.mod.json
    data/
      terrain.json
      terrain_visuals.json
      autoplace.json
      planet_presets.json
    textures/
      terrain/
        red_sand/
          base/1x1.png

Relative paths inside content files resolve from the mod folder. Absolute paths, res://, and user:// are accepted.

内容文件里的相对路径会从 mod 文件夹开始解析。也可以使用绝对路径、res:// 和 user://。

Loading Rules

RuleBehavior
Corecore always loads first and cannot be disabled.
Active modsThe Mods screen writes user://mods/active_mods.json.
DependenciesDependencies load before dependents. Missing dependency disables the dependent mod.
OrderAfter dependencies, user order decides load order.
OverridesIf two mods define the same content ID, the later mod replaces the entire definition.
ReloadMods reload from the main menu. Worlds do not hot-reload mods while running.

加载规则

规则行为
Corecore 永远最先加载，不能禁用。
启用列表Mods 界面会写入 user://mods/active_mods.json。
依赖依赖项会先于依赖它的 mod 加载。缺少依赖时，相关 mod 不会进入 registry。
顺序依赖顺序确定后，同级 mod 按用户排序加载。
覆盖两个 mod 定义同一个内容 ID 时，后加载的 mod 会整条替换前一个定义。
重载mod 只在主菜单重载。进入世界后不做热重载。

Manifest

The manifest identifies the mod and points to content files.

Manifest

manifest 用来声明 mod 身份，并指向它提供的内容文件。

{
  "schema_version": 1,
  "id": "author.my_mod",
  "name": "My Mod",
  "version": "1.0.0",
  "game_api": "0.1",
  "dependencies": [
    { "id": "core", "version": ">=0.1.0" }
  ],
  "content": {
    "terrain": ["data/terrain.json"],
    "terrain_visuals": ["data/terrain_visuals.json"],
    "autoplace": ["data/autoplace.json"],
    "planet_presets": ["data/planet_presets.json"]
  }
}
FieldRequiredDescription
idYesStable mod ID. Use lowercase namespace format like author.mod_name.
versionYesSemantic version string. Saves record exact active mod versions.
game_apiYesMust be 0.1 for this API version.
dependenciesNoSupports exact versions and >=x.y.z ranges.
contentYesLists JSON files for supported content domains.
字段必填说明
id是稳定的 mod ID。推荐使用小写命名空间格式，例如 author.mod_name。
version是语义化版本字符串。存档会记录启用 mod 的精确版本。
game_api是这个 API 版本必须填写 0.1。
dependencies否支持精确版本和 >=x.y.z 范围。
content是列出这个 mod 提供的各类 JSON 内容文件。

Terrain

Terrain IDs are stable content IDs. Runtime integer IDs are assigned by the game and saved through a palette.

地形

地形 ID 是稳定内容 ID。运行时整数 ID 由游戏分配，并通过 palette 写入存档，不应该当成长期内容 ID 使用。

[
  {
    "id": "author.my_mod.red_sand",
    "aliases": ["red_sand"],
    "display_name": "Red Sand",
    "map_color": "0xb45a3a",
    "walkable": true,
    "buildable": true,
    "is_water": false
  }
]
FieldDescription
idStable terrain ID. Prefer mod_id.local_name.
aliasesOptional short names for compatibility or convenience.
preferred_runtime_idOptional hint. Do not rely on it for save compatibility.
map_colorMinimap color as integer or 0xRRGGBB.
walkable/buildable/is_waterGameplay flags used by movement, landfill, collision, and generation.
字段说明
id稳定地形 ID。推荐使用 mod_id.local_name。
aliases可选短名，用于兼容旧数据或方便内部引用。
preferred_runtime_id可选提示值。不要依赖它做存档兼容。
map_color小地图颜色，可以是整数或 0xRRGGBB。
walkable/buildable/is_water玩法标记，用于移动、填海、碰撞和生成逻辑。

Terrain Visuals

Visual definitions connect terrain IDs to terrain atlas textures. base_1x1 is required. Other textures are optional and skipped when absent.

地形视觉

视觉定义把地形 ID 连接到地形图集贴图。base_1x1 是必需贴图，其他贴图缺失时会跳过对应图层。

[
  {
    "terrain_id": "author.my_mod.red_sand",
    "priority": 35,
    "is_water": false,
    "textures": {
      "base_1x1": "textures/terrain/red_sand/base/1x1.png",
      "base_2x2": "textures/terrain/red_sand/base/2x2/2x2.png",
      "base_4x4": "textures/terrain/red_sand/base/4x4/4x4.png",
      "overlay": "textures/terrain/red_sand/overlay/dual16.png"
    }
  }
]
TextureExpected Sheet
base_1x11024x64, 16 horizontal 64x64 variants.
base_2x22048x128, 16 horizontal 128x128 variants.
base_4x44096x256, 16 horizontal 256x256 variants.
overlay1024x1024, 16 mask columns by 16 cycle rows.
shore_shadowOptional land-specific water edge shadow sheet.
shoreOptional land-specific water edge rim sheet.
shore_combinedOptional combined shore sheet that replaces separate shore layers when present.
animated_*Optional path format for water animation frames, using {0} as the frame number.
foam_textureOptional top-layer water edge sheet for water terrains.
贴图字段图集要求
base_1x11024x64，横向 16 个 64x64 变体。
base_2x22048x128，横向 16 个 128x128 变体。
base_4x44096x256，横向 16 个 256x256 变体。
overlay1024x1024，16 列 mask 乘 16 行 cycle。
shore_shadow可选，陆地侧水岸阴影图集。
shore可选，陆地侧水岸边缘图集。
shore_combined可选，合并水岸图集；存在时优先于拆分的 shore 图层。
animated_*可选，水体动画帧路径格式，使用 {0} 作为帧号占位。
foam_texture可选，水体地形的顶层水岸泡沫图集。

Autoplace

Autoplace rules score terrain candidates from height, moisture, temperature, and variation noise. The highest score wins.

自动生成规则

自动生成规则会根据高度、湿度、温度和变化噪声为地形候选项打分。得分最高的地形会被放置。

[
  {
    "id": "author.my_mod.red_sand",
    "terrain_id": "author.my_mod.red_sand",
    "height_range": [-0.2, 0.65],
    "moisture_range": [-1.0, 0.05],
    "temperature_range": [0.2, 1.0],
    "base_score": 0.16,
    "height_weight": 0.35,
    "moisture_weight": 0.70,
    "temperature_weight": 0.35,
    "variation_weight": 0.06,
    "range_falloff": 0.42
  }
]

Planet Presets

A planet preset controls which terrain IDs are enabled for a new world. Modded terrain will not generate unless the selected preset includes it.

星球预设

星球预设决定新世界启用哪些地形 ID。一个 mod 地形只有被当前预设包含时，才会参与地图生成。

[
  {
    "id": "author.my_mod.red_desert",
    "aliases": ["red_desert"],
    "label": "Red Desert",
    "description": "Dry world with red sand.",
    "terrain_ids": [
      "core.base_soil",
      "core.dirt",
      "author.my_mod.red_sand",
      "core.water",
      "core.deep_water"
    ],
    "surface_properties": {
      "gravity": 10,
      "pressure": 930,
      "magnetic_field": 80,
      "solar_power": 120,
      "day_night_cycle": 420
    }
  }
]

Save Compatibility

When a planet is created, the save records active mod IDs, versions, load order, a game data fingerprint, and a terrain palette. Loading is strict.

ProblemResult
Required mod missingLoad is blocked.
Version mismatchLoad is blocked.
Extra mod activeLoad is blocked.
Terrain palette ID missingLoad is blocked.
Legacy save with core onlyAllowed when no non-core mods are active.

存档兼容

创建星球时，存档会记录启用的 mod ID、版本、加载顺序、游戏数据 fingerprint 和地形 palette。读档会严格校验。

问题结果
必需 mod 缺失阻止读档。
版本不一致阻止读档。
额外 mod 启用阻止读档。
地形 palette ID 无法解析阻止读档。
只有 core 的旧存档没有启用非 core mod 时允许读取。

Example Mod

A copyable example lives in docs/modding/examples/simple_terrain_mod. It reuses core sand textures so it can load without custom art.

示例 Mod

可复制的示例位于 docs/modding/examples/simple_terrain_mod。它复用了 core 的沙地贴图，所以没有自定义美术也能加载。

docs/modding/examples/simple_terrain_mod/
  factoria.mod.json
  data/
    terrain.json
    terrain_visuals.json
    autoplace.json
    planet_presets.json

Copy that folder to user://mods/example.red_sand, enable it in the Mods screen, reload, and choose the Red Desert planet preset.

把这个文件夹复制到 user://mods/example.red_sand，在 Mods 界面启用，重新加载后选择 Red Desert 星球预设。
