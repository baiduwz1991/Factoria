# FACTORIA 项目愿景

## 一句话定位

FACTORIA 是一个以异星工场式自动化生产为基础，并加入 agent coding 蓝图体验的工厂构建游戏。

玩家不仅手动摆放机器、传送带和物流网络，也可以像和编程助手协作一样，用自然语言驱动 agent 生成、修改、解释和优化工厂蓝图。

## 核心体验

### 1. 异星工场式自动化

项目的基础体验来自自动化生产链：

- 采集资源。
- 建造生产设备。
- 连接物流与电力。
- 扩展产线规模。
- 优化吞吐、瓶颈和空间布局。

这部分提供游戏的确定性规则、资源约束和工程感。

### 2. Vibe Coding 式蓝图协作

在传统蓝图系统之上，引入类似 agent coding 的交互方式：

- 玩家描述目标，agent 生成一份蓝图草案。
- 玩家提出修改意见，agent 根据约束迭代蓝图。
- 玩家框选一块已有工厂，agent 分析功能、瓶颈和可改造点。
- 玩家把分析结果转成新蓝图，继续复用、改造或扩建。

这部分提供“和智能助手一起设计工厂”的新鲜体验。

## 设计目标

1. 让玩家从“摆放单个建筑”扩展到“描述意图并审阅方案”。
2. 让蓝图不只是复制粘贴工具，而是可被解释、修改和推理的设计对象。
3. 让 agent 的输出受游戏规则约束，避免变成无成本的作弊建造。
4. 让玩家始终保留最终决策权，agent 负责提出方案、说明理由和执行可验证修改。

## 非目标

当前阶段不追求完整复刻异星工场。

早期重点应放在一个最小闭环：

- 有基础建筑与网格。
- 有可序列化蓝图。
- 有蓝图放置、预览和应用。
- 有一个 agent 输入入口。
- 有一个能被 agent 读取与修改的蓝图数据结构。

复杂科技树、敌人、多人联机、真实 LLM 接入成本控制等内容可以后置。

## 玩家想象场景

### 生成蓝图

玩家输入：

> 帮我做一条铁板产线，输入铁矿，输出铁板，尽量紧凑，留出向右扩展空间。

agent 输出：

- 一份可预览蓝图。
- 所需建筑与资源清单。
- 输入输出方向说明。
- 设计理由与限制。

### 修改蓝图

玩家输入：

> 把这条产线改成双倍产能，但不要超过当前宽度。

agent 输出：

- 修改后的蓝图差异。
- 产能变化说明。
- 如果无法满足约束，给出冲突原因。

### 框选分析

玩家框选已有工厂区域后输入：

> 分析这里为什么铁板不够。

agent 输出：

- 当前区域的设备、物流和连接摘要。
- 可能瓶颈。
- 可选优化方案。
- 是否建议转成可复用蓝图。

## 项目气质

FACTORIA 的重点不是让 agent 替玩家玩游戏，而是让玩家体验一种新的工厂设计方式：

玩家提出意图，agent 给出工程草案；玩家审阅、调整、否决或采纳。游戏乐趣来自规则约束下的人机协作设计。
